API9:2023 - Improper Inventory Management

APIs tend to expose more endpoints than traditional web applications, making proper and updated documentation highly important. A proper inventory of hosts and deployed API versions also are important to mitigate issues such as deprecated API versions and exposed debug endpoints.

LAB Scenario

API 9
Context:

A large organization develops and uses various internal APIs for different departments. There's no centralized registry or documentation for all APIs within the organization.

Improper Inventory Management:

The organization lacks a proper inventory of APIs. Many APIs are undocumented, and their existence is only known to a small group within specific departments. No clear ownership or lifecycle management is established for these APIs.

Attack Scenario:
  1. Internal Actor Negligence (Unintentional): A developer working on a new project stumbles upon an undocumented API endpoint while investigating the network traffic. Due to lack of documentation, the purpose and security posture of the API are unclear.
  2. Accidental Misuse: Without proper understanding, the developer decides to integrate with the undocumented API in their new project. This could lead to unintended consequences:
    • Data Exposure: The undocumented API might expose sensitive data that the new project shouldn't access.
    • Broken Functionalities: Integrating with an outdated or unsupported API could lead to compatibility issues or unexpected behavior in the new project.
Impact:

Improper inventory management can lead to several negative consequences:

  • Security Risks: Undocumented APIs create blind spots, making them vulnerable to exploitation by attackers. These attackers might be external or even malicious actors within the organization.
  • Integration Issues: Integrating with undocumented APIs can lead to development delays and compatibility problems in new projects.
  • Compliance Issues: Lack of API inventory makes it difficult to demonstrate compliance with security regulations or internal governance policies.

Mitigation

  • Implement a centralized API registry to document all APIs, including their purpose, functionalities, ownership, and access controls.

  • Enforce clear API lifecycle management practices, including development, deployment, versioning, deprecation, and retirement of APIs.

  • Foster a culture of API awareness within the organization. Encourage developers to document their APIs and report any undocumented APIs they encounter.

  • Regularly review and update the API inventory to ensure its accuracy and completeness.
